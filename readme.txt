Часть 1. Работа с пользователем

1) Создание системы регистрации пользователя в проекте (кол-во информации о пользователе не важно)
2) Создание системы авторизации пользователя в проекте
-=-=-
- Поле Name: допускает значение вида Ххх и более букв нижнего регистра англ алфавита, первая - Заглавная!
- Поле Email: допускает значение вида хх@xx.xx и более букв нижнего регистра англ алфавита
- При выходе из полей осуществляется проверка и кнопка Sign in становится доступной
- Запрос на авторизацию происходит средствами js(get-запрос в базу), если пользователь найден -
- ему добавляется кука(user=Имя)+перезагрузка. Если email не совпал - в консоле ответ.
- Если такого пользователя ещё в базе нет, отправляем запрос в базу на создание пользователя
- и "корзины"-со-списком-друзей +перезагрузка.
-=-=-
3) Создание закрытой части проекта, доступной только после авторизации
4) Незарегистрированным пользователям доступна только страница регистрации/авторизации.
-=-=-
- Если у пользователя есть кука(user) и она не пустая(стёртая) на её основе отрисовывается основная часть
- Основная часть:
- -- Область пользователя:
- -- Имя ----------- Email
- -- НаГлавной ----- Выйти
--------------------------
- список карточек пользователей пожелавших остаться на гланой(по умолчанию)
- карточка друга содержит Имя и один из четырёх наборов кнопок:
-
- 1. Синяя - подать заявку(Новый пользователь), переходите на пункт ниже
- 2. Желтая - вы отправили другу заявку, переходите на пункт выше
-
- 3. Зелёная - вы принимаете заявку от друга, переходите на пункт ниже
-    Серая - вы отклоняете завку друга, переходите на первый пункт
- 4. Красная - вы приостанавливаете дружбу, переходите на пункт выше
-=-=-


Часть 2. Создание контактов в системе (общие и личные)
1) Необходимо придумать и разместить на главной странице проекта произвольный «общий список контактов»,
который будет доступен каждому авторизованному пользователю в системе.
(достаточно заполнить 10-15 произвольных контактов)
2) Добавить возможность сохранения в личные контакты пользователя контактов из общего 
списка (раздел избранное)
-=-=-
- Email друга видно только когда дружба согласована(личные данные)
- 
- База данных Firebase. Запись по ключу. Чтение открыто.

- прикладываю фото структуры (db.jpg)

Результатом выполнения всех частей ТЗ должны стать
- Система авторизации/регистрации пользователя и страница авторизации(регистрации)
- Главная страница с общим списком контактов (10-15 контактов) - должна
отображаться одинаково для всех авторизованных пользователей
- Раздел избранных контактов пользователя - должен быть разным для
каждого авторизованного пользователя.

Формат результатов и сроки

Результаты принимаются в виде архива с исходным кодом, либо ссылки на
публичный репозиторий (github или другой ресурс). Необходимы файлы
миграции Базы данных.

В  случае если использовались какие-либо фреймворки необходимо указать
какие именно и предоставить инструкции по запуску проекта.
